## Priority notes for this domain

1. One project in "almost shipped" state → finishing beats starting; ship it first.
2. A project untouched for 4+ weeks → decide explicitly: resume, pause, or abandon. A conscious "abandoned" line in history is a win, not a failure; zombie projects tax every review.
3. Next steps must stay session-sized — if a next step has survived three reviews, it's too big; split it.
4. All healthy → spend the budget on the project whose goal-gap is smallest (momentum compounds).
